# Small Village — 기획 및 기능 추천 제안서 (Feature Recommendations)

본 문서는 `Small Village`를 업계 선두 가상 오피스 서비스(Gather, Kumospace, Tandem 등)들과 차별화하고, 사용자 경험(UX)을 대폭 고도화하기 위한 기획 및 추천 기능 제안서입니다. 날짜별로 고유한 내용들을 일목요연하게 정리하고 중복을 배제하여 작성되었습니다.

---

## 📅 2026-07-16 (금일 기획 및 차세대 피처 추천)

### 1. 아바타 머리 위 "Presence Indicator" (실시간 상태 표시)
* **제안 배경:** 가상 오피스에서 가장 흔히 일어나는 커뮤니케이션 피로감은 "상대방에게 말을 걸어도 되는지(자리에 있는지)"의 불확실성에서 옵니다. 타사(Gather, Tandem, Pragli)는 상태 플래그를 정교하게 지원하여 이를 해결하고 있습니다.
* **기획 내용:** 아바타 스프라이트 근처 혹은 닉네임 옆에 작은 상태 아이콘 배지를 실시간 표시합니다.
  - **상태 종류:** 🟢 업무 가능(Available), 🟡 자리 비움(AFK - 5분간 키보드/마우스 무반응 시 자동 전환), 🔴 방해 금지(Busy - Focus 모드 작동 중).
  - **기술적 구현 (Serverless 구조 정합):** 
    - `users` 테이블에 `presence_status` 컬럼을 열거형(ENUM)으로 추가합니다.
    - 유저가 바텀바나 상태 레이아웃에서 상태를 전환하면 `users` 테이블을 upsert하고, 다른 유저들은 기존 `postgres_changes` 구독 흐름에 맞춰 Phaser 스프라이트 옆에 배지(작은 동그라미 텍스처 혹은 이모지)를 업데이트하도록 설계합니다. 추가 프로세스 없이 초경량으로 구현 가능합니다.

### 2. 공간 오디오 전송 범위 인디케이터 (Proximity Radius Indicator)
* **제안 배경:** Cloudflare RealtimeKit 기반의 공간 오디오(Spatial Audio)는 가까워질수록 소리가 들리는 핵심 가치이나, 사용자는 "내가 저 사람과 얼마나 가까워져야 내 목소리가 들리는지"를 직관적으로 알기 어렵습니다.
* **기획 내용:** 내 캐릭터를 중심으로 소리가 닿는 임계 영역(예: 반경 120px)을 바닥에 아주 옅고 부드러운 반투명 원형(Proximity Guide Ring)으로 렌더링해 줍니다.
  - **UX 장점:** 이 원 영역 안으로 다른 캐릭터가 진입하면 "아, 이제 음성이 연결되겠구나"를 시각적으로 단번에 인지할 수 있게 됩니다.
  - **기술적 구현:** Phaser 내에서 캐릭터가 생성되는 컨테이너 밑에 `Phaser.GameObjects.Graphics`를 사용하여 투명도 0.1~0.15 수준의 원형 도형을 하나 그리고, 플레이어 이동 이벤트가 발생할 때 함께 따라다니도록 처리합니다. GPU 부하가 거의 없는 가벼운 렌더링입니다.

### 3. 무공백 URL 하이퍼링크 파서 & 텍스트 원클릭 복사
* **제안 배경:** 게임 씬 내부 말풍선에 올라온 기획서 링크, 노션 문서 주소 등 긴 URL을 사용자들이 마우스로 긁어 복사하거나 클릭하기가 극히 어렵습니다.
* **기획 내용:** 인게임 말풍선 내의 텍스트 중 `http://` 또는 `https://` 패턴을 감지하여, 해당 링크 영역을 클릭 시 즉시 클립보드에 복사되거나 브라우저 새 탭으로 여는 액션을 바인딩합니다.
  - **기술적 구현:** `src/scenes/bubbleText.ts` 에 말줄임 가공 외에, 링크를 감지하여 해당 URL을 별도로 추출하는 정규식 도우미를 추가합니다. Phaser 말풍선 컨테이너 내부에 링크가 감지되었을 경우 작은 "🔗" 모양 아이콘 버튼을 띄우고, 이 아이콘 오브젝트에 `.setInteractive({ useHandCursor: true })` 및 `pointerdown` 리스너를 매핑해 `window.open(url)` 이 실행되도록 돕습니다.

### 4. 접근성(A11y) 보완을 통한 테스트 자동화 기반 강화 (QA 친화적 UI 개선)
* **제안 배경:** 현재 Playwright를 통한 E2E 테스트 시 바텀바 버튼들이 정상 렌더링됨에도 탐색에 실패해 오탐을 유발하고 있습니다.
* **기획 내용:** 바텀바 내의 UI 요소들(`IconButton` 등)에 대해 시맨틱 웹 표준을 준수하고 고유한 테스트 셀렉터를 명시합니다.
  - **구현 방식:**
    - 모든 커스텀 이미지/아이콘 클릭 요소를 `<button>` 태그로 감싸고 `aria-label="Toggle Microphone"`, `aria-label="Open Chat"` 등의 설명을 명시합니다.
    - 개발/QA 단에서 접근하기 용이하도록 엘리먼트에 `data-testid="bottom-mic-btn"` 와 같은 고유 속성을 부여하여 렌더링 엔진의 레이아웃 변경이 있어도 자동화 스크립트가 흔들리지 않고 동작하게 합니다.

## 📅 2026-07-15 (이전 기획 회의 아카이브)

### 1. 텔레그램 기반 실시간 유저 입장 알림 (Supabase Edge Function 연동)
* **기획 배경:** 소규모 협업용 공간 특성상, 누군가 공간에 들어왔을 때 알림을 받지 못하면 가상 오피스에 하루 종일 무의미하게 서 있게 됩니다.
* **구현 설계:** 
  - 유저가 입장하여 `users` 테이블에 INSERT/upsert가 완료되는 즉시, Supabase의 `supabase.functions.invoke('notify-telegram', { body })` 를 트리거합니다.
  - Edge Function(`notify-telegram`) 내에서 Deno 런타임 환경변수(`TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID`)를 바탕으로 텔레그램 Bot API를 호출하여 지정된 채널이나 본인 DM으로 `"DailyQA님이 'Lobby'에 입장했습니다!"` 알림을 전송합니다.
  - **차별점:** 별도의 24/7 백그라운드 봇 프로세스나 폴링 서버 없이, 100% 서버리스(Serverless) 온디맨드 형태로 구동되므로 유지비용이 0원에 수렴하며 완벽하게 안정적입니다.

### 2. 음성 발화 하이라이트 "스피커 링 (Speaker Ring)"
* **기획 배경:** 누가 말하고 있는지 알 수 없어 여러 명이 모였을 때 혼선이 생기는 문제를 해결합니다.
* **구현 내용:** 
  - Cloudflare RealtimeKit의 `speakingPeers` 이벤트를 구독하여 발화가 탐지된 유저 ID(`customParticipantId`)를 식별합니다.
  - 해당 유저의 Phaser 아바타 스프라이트 바닥에 녹색의 부드러운 스피커 원형 링(또는 머리 위 스피커 아이콘)을 발화하는 동안 주기적으로 펄스(Pulse) 애니메이션 형태로 노출합니다.
  - **달성 완료:** 해당 기능은 `src/components/SpeakerIndicators.tsx` 및 `useSpeakingPeers.tsx`를 통해 1차 MVP 적용이 성황리에 머지 완료되었습니다.

### 3. 실시간 방 인원수 배지 (Room Counts Dashboard)
* **기획 배경:** 첫 화면 로비에서 어떤 방에 사람이 많이 모여 있는지 파악한 후 입장하기를 원하는 유저들의 니즈를 반영합니다.
* **구현 내용:** 
  - 방 목록 화면에서 방 이름 우측에 실시간으로 접속해 있는 동시 접속자 수 배지(`(2/10)`)를 노출합니다.
  - **달성 완료:** Supabase Presence 및 `roomCounts.ts`를 연동하여 실시간 동시 접속자 수가 반영되도록 구현을 마쳤습니다.
